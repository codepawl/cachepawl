\begin{abstract}
Hybrid language models like Jamba mix attention layers with State Space Models (SSMs), forcing inference engines to manage two memory profiles simultaneously: Key-Value (KV) caches that grow linearly with sequence length, and fixed-size SSM states. Current allocators handle this poorly. Unified pools (vLLM-style) pad SSM states to attention page sizes, wasting up to $7.3\times$ capacity. Static dual pools (SGLang-style) cannot adapt when prompt distributions shift between requests.

We present Asymmetric Virtual Memory Paging (AVMP), an allocator that separates the two cache types into physically distinct backing stores behind a unified virtual address space, and dynamically migrates capacity between them when one pool approaches saturation. AVMP triggers migration strictly inside CapacityError handlers, ensuring deterministic behavior without speculative rebalancing.

We evaluate AVMP across 270 cells of synthetic workloads plus a 60-cell ShareGPT trace replay on an RTX 3060 12GB. The dynamic allocator reduces Out-of-Memory events by 7.6\% (95\% CI $[3.0\%, 12.7\%]$) and improves goodput by $1.83\times$ to $13.3\times$ across synthetic workloads and $2.36\times$ on ShareGPT. A phase-time breakdown identifies two distinct mechanisms: shorter OOM recovery on capacity-pressured workloads, and faster per-call allocation on KV-heavy workloads. We release a Python reference implementation with deterministic reproducibility for event-level metrics.
\end{abstract}
