## Decision

Project work should be divided into small sprints with clear outputs and definitions of done.

## Reason

Small increments reduce scope drift and make agent-generated changes easier to review.

## Consequences

- Agents should only work on the current sprint unless explicitly instructed.
- Future-sprint work should remain in backlog or TODO sections.
- Sprint details live in one file per sprint so indexes stay short.
